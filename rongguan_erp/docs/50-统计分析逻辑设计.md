# W70 统计分析逻辑设计文档

## 一、概述

本文档描述如何从 W70 Sample Sales Base 基础数据中统计出前端 Dashboard（`patten-sales-ai`）所需的各种分析数据。

### 1.1 数据流向

```
ERPNext (W70 Sample Sales Base 基础数据)
    ↓
FastAPI 统计服务（数据聚合和计算）
    ↓
前端 Dashboard（客户分析、款式分析、业务对比、预警监控）
```

### 1.2 核心概念

- **基础数据**：W70 Sample Sales Base 表中的原始记录
- **打样数据**：`sheet_type = '打样'` 的记录（开发成本）
- **销样数据**：`sheet_type = '销样'` 的记录（订单利润）
- **统计维度**：客户、款式、业务类型、时间等

---

### 2.1 关键字段

| 字段名 | 类型 | 说明 | 用途 |

### 2.2 数据分类

**打样数据（开发成本）**：
- `sheet_type = '打样'`
- 用于统计开发成本
- 关键字段：`total_cost` 或各项费用总和

**销样数据（订单利润）**：
- `sheet_type = '销样'`
- 用于统计订单利润
- 关键字段：`gross_profit` 或 `amount - total_cost`

## 三、统计维度与指标

### 3.1 客户分析统计

#### 3.1.1 输入数据

- **打样记录**：`sheet_type = '打样'` 的所有记录
- **销样记录**：`sheet_type = '销样'` 的所有记录

#### 3.1.2 统计逻辑

**按客户分组统计：**

```python
# 伪代码
for customer in all_customers:
    # 1. 统计打样数据（开发成本）
    development_records = filter(打样数据, customer=customer)
    total_development_cost = sum(development_records.total_cost)
    development_style_numbers = unique(development_records.style_number)
    development_style_count = len(development_style_numbers)
    
    # 2. 统计销样数据（订单利润）
    sales_records = filter(销样数据, customer=customer)
    total_order_profit = sum(sales_records.gross_profit)
    order_count = len(sales_records)
    successful_style_numbers = unique(sales_records.style_number)
    successful_style_count = len(successful_style_numbers)
    
    # 3. 计算指标
    coverage_rate = (total_order_profit / total_development_cost) * 100  # 开发费用覆盖率
    success_rate = (successful_style_count / development_style_count) * 100  # 成功率
    average_return_per_style = total_order_profit / development_style_count  # 单款平均回报
    roi_rating = calculate_roi_rating(coverage_rate)  # ROI评级
```

#### 3.1.3 输出数据结构

```typescript
interface CustomerMetrics {
  customer: string                    // 客户名称
  businessType: '外贸' | '内销'        // 业务类型（从数据中获取）
  totalDevelopmentCost: number        // 累计开发成本
  totalOrderProfit: number            // 累计订单利润
  coverageRate: number                // 开发费用覆盖率（%）
  successRate: number                 // 样衣开发成功率（%）
  roiRating: 'A' | 'B' | 'C' | 'D'   // 投资回报评级
  averageReturnPerStyle: number       // 单款样衣平均回报
  developmentStyleCount: number       // 打样款号数
  successfulStyleCount: number        // 成功款号数
  orderCount: number                  // 订单数量
  lastOrderDate?: Date                // 最后订单时间
}
```

#### 3.1.4 计算公式

1. **开发费用覆盖率**：
   ```
   coverage_rate = (累计订单利润 / 累计开发成本) × 100
   ```

2. **样衣开发成功率**：
   ```
   success_rate = (成功款号数 / 打样款号数) × 100
   ```

3. **投资回报评级**：
   ```
   if coverage_rate >= 200: 'A'
   else if coverage_rate >= 150: 'B'
   else if coverage_rate >= 100: 'C'
   else: 'D'
   ```

4. **单款样衣平均回报**：
   ```
   average_return = 累计订单利润 / 打样款号数
   ```

## 六、数据过滤逻辑

### 6.1 时间范围过滤

```python
def filter_by_time_range(
    data: List[W70SampleSalesBase],
    time_range: str,
    start_date: Optional[date] = None,
    end_date: Optional[date] = None
) -> List[W70SampleSalesBase]:
    if time_range == 'custom':
        if start_date and end_date:
            return [d for d in data if start_date <= d.contract_date <= end_date]
    elif time_range == '3m':
        cutoff_date = date.today() - timedelta(days=90)
        return [d for d in data if d.contract_date >= cutoff_date]
    elif time_range == '6m':
        cutoff_date = date.today() - timedelta(days=180)
        return [d for d in data if d.contract_date >= cutoff_date]
    elif time_range == '12m':
        cutoff_date = date.today() - timedelta(days=365)
        return [d for d in data if d.contract_date >= cutoff_date]
    elif time_range == 'year':
        current_year = date.today().year
        return [d for d in data if d.contract_date.year == current_year]
    
    return data
```

## 七、性能优化建议

### 7.1 数据缓存

- **缓存策略**：对统计结果进行缓存，减少重复计算
- **缓存时间**：根据数据更新频率设置缓存过期时间（建议 5-10 分钟）
- **缓存键**：基于筛选参数生成唯一缓存键

### 7.2 数据库查询优化

- **索引优化**：确保 `customer`、`style_number`、`business_type`、`contract_date` 等字段有索引
- **批量查询**：一次性查询所有需要的数据，避免多次查询
- **数据预聚合**：对于常用的统计维度，可以考虑在数据库层面进行预聚合

### 7.3 分页和限制

- **分页查询**：对于大量数据，支持分页查询
- **结果限制**：限制返回的记录数量，避免一次性返回过多数据
