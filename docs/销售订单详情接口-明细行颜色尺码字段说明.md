# 销售订单详情接口：明细行颜色、尺码字段说明

**读者**：前端工程师  
**更新日期**：2026-03-13  
**接口**：`/api/method/rongguan_erp.utils.api.sales_order.get_sales_order_detail`

---

## 一、说明

后端已在「销售订单详情」接口的**每条明细（item）**中返回 **`custom_color`**、**`custom_size`** 两个字段，前端可直接使用，无需再从 `item_code` / `item_name` / `description` 做文本解析。

---

## 二、返回字段（每条 item 必有）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| **custom_color** | string | 该明细行的颜色。来源于 Item 变体属性中「颜色」类型属性（Item Attribute 的 _user_tags 含「颜色」）的 attribute_value。无则返回 `""`。 |
| **custom_size** | string | 该明细行的尺码。来源于 Item 变体属性中「尺码」类型属性（Item Attribute 的 _user_tags 含「尺寸」）的 attribute_value。无则返回 `""`。 |

- 每条 item 都会带上这两个 key，不会缺失；没有值时为空字符串 `""`。
- 取值逻辑：后端只从 **Item Variant Attribute + Item Attribute 的 _user_tags** 取，不做任何从 item_code/item_name/description 的解析。

---

## 三、使用建议

- 在裁剪报工等依赖「销售订单明细颜色/尺码」的页面，**优先读取** `item.custom_color`、`item.custom_size`。
- 仅当这两个字段都为空时，再使用你们现有的兜底逻辑（若有）。

---

## 四、返回示例（items 中单条）

```json
{
  "item_code": "10910599-110103 Egret 米色-44",
  "item_name": "10910599 (110103 Egret 米色, 44)",
  "description": "10910599 (110103 Egret 米色, 44)",
  "qty": 1,
  "custom_color": "110103 Egret 米色",
  "custom_size": "44"
}
```

---

**总结**：接口已稳定返回 `custom_color`、`custom_size`，前端可直接使用这两个字段展示颜色与尺码。
